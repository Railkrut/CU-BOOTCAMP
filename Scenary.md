# Сценарий пользовательского опыта:


Формат обучающего сайта для стартапов и решения кейсов с кейс-чемпионатов у студентов

Фронтенд в формате SPA, перход между страница происходит за счёт изменения url,
переход между страниц за счёт формата (адрес-сайта/#/route)

(frontend/src/pages/main.jsx)
## Главный экран при входе. 
- 1 поле - идея стартапа
- 2 поле - некие доп данные для уточнения идей
- окно, в котором ИИ выводит основные риски проекта
- кнопка "Начать обучение", перевод пользователя на квиз

## Формат квиза.

На основе данных полученных у пользоваетля ИИ формирует вопросы для проверки адеватности идеи(в это не углубляться).
Каждая страница квиза имеет:
- вопрос, полученнный на основе данных, пришедших с бэкенда
- диалоговое окно с полем введения ответа на вопрос textarea
- окно выведения данных полученных с бекенда на основе ответа на вопрос пользователя(рекомендации к решению вопроса).

Квиз имеет n кол-во вопросов.

## Заключительная страница.

Итоговая оценка данных, собранных с пользователя, для создания заказа
